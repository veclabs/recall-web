## What makes VecLabs different

Most vector databases are centralized infrastructure you rent access to. Your data lives on their servers. You trust their uptime, their pricing, and their word that nothing has changed.

VecLabs is built differently across three dimensions:

<CardGroup cols={3}>
  <Card title="Speed" icon="gauge">
    **4.7ms p99** at 100K vectors, 1536 dimensions. Rust HNSW in-process — no network round-trip, no garbage collector.
  </Card>
  <Card title="Privacy" icon="lock">
    Vectors are encrypted with **AES-256-GCM** before leaving the SDK. VecLabs cannot read your data.
  </Card>
  <Card title="Verifiability" icon="link">
    After every write, a **32-byte Merkle root** is posted to Solana. Anyone can verify your data hasn't been tampered with.
  </Card>
</CardGroup>

## Your first query in 30 seconds

<CodeGroup>
```typescript TypeScript
import { SolVec } from '@veclabs/solvec';

const sv = new SolVec({ network: 'devnet' });
const collection = sv.collection('my-collection', { dimensions: 1536 });

// Store a vector
await collection.upsert([{
  id: 'doc_001',
  values: Array(1536).fill(0).map(() => Math.random()),
  metadata: { text: 'Hello from VecLabs' }
}]);

// Query it back
const results = await collection.query({
  vector: Array(1536).fill(0).map(() => Math.random()),
  topK: 5
});

console.log(results);
```

```python Python
from solvec import SolVec

sv = SolVec(network="devnet")
collection = sv.collection("my-collection", dimensions=1536)

# Store a vector
collection.upsert([{
    "id": "doc_001",
    "values": [0.1] * 1536,
    "metadata": {"text": "Hello from VecLabs"}
}])

# Query it back
results = collection.query(
    vector=[0.1] * 1536,
    top_k=5
)

print(results)
```
</CodeGroup>

## Current status

VecLabs is currently in **alpha**. The API surface is stable and will not change. Some backend features are still in progress.

| Component | Status |
|---|---|
| Rust HNSW core | ✅ Complete — 31 tests, 2.0ms p99 at 100K vectors |
| AES-256-GCM encryption | ✅ Complete |
| Merkle tree + proof generation | ✅ Complete |
| Solana Anchor program | ✅ Live on devnet |
| TypeScript SDK | ✅ Alpha — `npm install @veclabs/solvec` |
| Python SDK | ✅ Alpha — `pip install solvec --pre` |
| Shadow Drive persistence | 🔄 In progress |
| WASM Rust bridge | 🔄 In progress |
| LangChain integration | 📋 Planned |
| Mainnet deployment | 📋 Planned |

<Note>
  During alpha, vectors are held in memory. Persistence to Shadow Drive is in active development. The demo resets on server restart — this will be resolved before mainnet.
</Note>
