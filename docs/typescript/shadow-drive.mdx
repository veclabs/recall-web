# Shadow Drive

Shadow Drive is Solana's decentralized storage protocol. When enabled,
VecLabs uploads your encrypted collection to Shadow Drive after every write —
creating a permanent, decentralized backup that survives server restarts.

---

## How it works

After every `upsert()` or `delete()`, VecLabs:

1. Writes the encrypted `.db` file to disk (synchronous — already happens in Phase 4)
2. Uploads the same encrypted buffer to Shadow Drive (async, fire-and-forget)

Shadow Drive **never blocks `upsert()`**. If Shadow Drive is unavailable,
the operation completes normally. No error is thrown.

---

## Enable Shadow Drive

```typescript
import { SolVec } from '@veclabs/solvec'

const sv = new SolVec({
  network: 'devnet',
  walletPath: '/Users/you/.config/solana/id.json',
  shadowDrive: true,   // ← enable
})

const memory = sv.collection('agent-memory', { dimensions: 1536 })
// Shadow Drive initializes in the background
```

`shadowDrive: true` requires `walletPath`. Without a wallet, Shadow Drive
is disabled silently and a warning is logged.

---

## Cold start restore

When your server restarts on a new machine with no local `.db` file,
call `restoreFromShadowDrive()` to rebuild from Shadow Drive:

```typescript
const memory = sv.collection('agent-memory', { dimensions: 1536 })
const restored = await memory.restoreFromShadowDrive()

if (restored) {
  console.log('Memory restored from Shadow Drive')
} else {
  console.log('No Shadow Drive backup found — starting fresh')
}
```

---

## Privacy

Everything uploaded to Shadow Drive is the same AES-256-GCM ciphertext
already written to disk. Storage nodes see only encrypted bytes.
VecLabs cannot read your vectors. Shadow Drive cannot read your vectors.

---

## Error handling

Shadow Drive errors are caught silently. If upload fails:
- A warning is logged: `[SolVec] Shadow Drive upload failed for 'name': reason`
- The `upsert()` call returns normally
- Your in-memory index and disk persist are unaffected
- The `available` flag is set to `false` for future calls

```typescript
// This always resolves — Shadow Drive failure does not throw
await memory.upsert([{ id: 'vec1', values: [...], metadata: {} }])
```

## Storage costs

Shadow Drive charges approximately $0.05/GB/year.
At 1M vectors with 1536 dimensions: ~$0.025/month.

Each Merkle root post to Solana: $0.00025.
